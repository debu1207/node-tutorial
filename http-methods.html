HTTP request methods

HTTP defines a set of request methods to indicate the desired action to be performed for a given resource.

Although they can also be nouns, these request methods are sometimes refereed to as HTTP verbs. Each of them implements a different semantic, but some common features are shared by a group of them: e.g. a request method can be safe, idempotent, or cacheable.

GET The GET method requests a representation of the specified resource. Requests using GET should only retrieve data.
HEAD The HEAD method asks for a reponse identical to a GET request, but without the response body.
POST The POST method submits an entity to the specified resource, often causing a change in state or side effects on the server.
PUT The PUT method replaces all current representations of the target resource with the request payload.
DELETE The DELETE method deletes the specified resource.
CONNECT The CONNECT method establishes a tunnel to the server identified by the target resource.
